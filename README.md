# todo_dev

todo_dev is an attempt at a complex command-line-interface parser.


Instead of structured input, the goal for this was unordered, varying, human-readable input. 


A few examples would look like:

$ todo "cat box" tomorrow at 6 15 PM

$ todo dishes in one hour

$ todo another_task at 10 PM on 3/11 warn 1 day


The project was fun but I stopped working on it for now. I might revisit it to see if I could get it working!
